# 2026春季 - Web 应用程序开发课程项目

## 项目目标
本项目旨在要求学生掌握 **SpringBoot 3.x** 与 **Vue3 / React / Next.js** 的全栈开发技能，并深度集成 **大语言模型 (LLM)**。学生需完成从需求分析、数据库设计、AI 提示词工程到云端自动化部署的完整生命周期。

---

## 示例主题

### 1. 智能知识库管理系统 (RAG-based Wiki)
* **AI 核心：** 支持用户上传 PDF/Markdown，利用向量数据库（Chroma / PGVector）实现基于文档内容的 **RAG 问答**。
* **功能：** 知识分级管理、语义搜索、AI 自动生成文档摘要。

### 2. 多智能体协作任务系统 (Agentic Task Manager)
* **AI 核心：** 用户输入模糊需求（如“帮我策划下周的团建”），AI 自动拆解为多个子任务并填充截止日期。
* **功能：** 任务自动分类、优先级智能建议、基于周报的 AI 生产力分析。

### 3. AI 驱动的代码评测与教学平台 (AI Tutor for Code)
* **AI 核心：** 集成 LLM 对学生提交的代码进行实时 **Code Review**，指出逻辑错误并给出修复建议（不直接给出答案，而是引导式提问）。
* **功能：** 在线编辑器、代码复杂度可视化、AI 自动生成单元测试用例。

### 4. 智能多维财务顾问 (AI Financial Advisor)
* **AI 核心：** AI 自动分类消费账单，基于历史数据预测下月支出，并提供个性化的省钱建议。
* **功能：** 语音记账（语音转文字+结构化提取）、多表联动统计、财务健康度评分。

### 5. AI 智能旅游规划专家 (AI Travel Planner)
*   **AI 核心：** 用户输入“我想去大理玩三天，预算3000，喜欢安静，不喜欢爬山”，AI 自动生成分日期的**结构化行程单**。
*   **Web 功能：**
    *   **交互地图：** 前端集成高德/Leaflet地图，将 AI 生成的景点坐标可视化。
    *   **预算追踪：** 自动计算行程预估开支，并与实际记账对比。
    *   **协同编辑：** 支持多人共同编辑同一个旅行计划。
*   **LLM 挑战：** 如何将 LLM 输出的非结构化文本解析为数据库可存储的 JSON 格式。

### 6. 智能多语言面试教练 (AI Mock Interviewer)
*   **AI 核心：** 用户上传简历和目标岗位，AI 模拟面试官通过**语音或文字**进行多轮追问。
*   **Web 功能：**
    *   **实时反馈：** 面试结束后，AI 生成详尽的评估报告（得分、回答亮点、改进建议）。
    *   **音视频处理：** 前端调用录音 API，后端对接语音转文字（STT）再送入 LLM。
    *   **历史趋势：** 记录多次模拟面试的得分曲线，分析进步情况。
*   **LLM 挑战：** 角色扮演（System Prompt）的稳定性控制。

### 7. AI 驱动的低代码 SQL 助手 (Natural Language to SQL)
*   **AI 核心：** 用户输入中文需求（如“统计去年消费最高的三个用户”），AI 自动生成 SQL 语句并执行，返回结果。
*   **Web 功能：**
    *   **动态报表：** 根据 SQL 返回的数据，前端动态渲染成折线图、饼图（ECharts）。
    *   **安全沙箱：** 后端需要严格校验 SQL 安全性，防止删库等注入攻击。
    *   **Schema 管理：** 用户可以上传 Excel/创建表结构，AI 自动学习表元数据。
*   **LLM 挑战：** 如何通过 Prompt 告知 LLM 数据库的 Schema 信息。

### 8. 个人知识库“大脑” (Second Brain with RAG)
*   **AI 核心：** 结合 **RAG (检索增强生成)** 技术。用户上传自己的笔记、论文或网页收藏，AI 只根据这些私有数据回答问题。
*   **Web 功能：**
    *   **向量化管理：** 后端集成 Vector Database (Chroma / PGVector)。
    *   **来源溯源：** AI 回答问题时，在 Vue / React / Next.js 界面上高亮显示参考了哪篇文档的哪一段。
    *   **多格式支持：** 支持 Markdown, PDF, Docx 的解析。
*   **LLM 挑战：** 文本分段（Chunking）策略与向量检索的准确度优化。

### 9. 智能健康与饮食管理系统 (AI Nutritionist)
*   **AI 核心：** 用户拍摄食物照片（或描述饮食），AI 识别食物成分并估算热量，给出当日营养建议。
*   **Web 功能：**
    *   **拍照识物：** 对接多模态模型（GLM-4V / Kimi K2.5 / DeepSeek-V4）。
    *   **健康档案：** 结合用户的身高、体重、运动量（SpringBoot 后端计算）生成个性化方案。
    *   **社交打卡：** 类似朋友圈的饮食分享社区。
*   **LLM 挑战：** 多模态（图片+文字）输入处理。

## 核心技术要求

### 1. 后端（Java / SpringBoot）
* **核心框架：** SpringBoot 3.2+（推荐使用 **Spring AI** 框架对接 LLM，简化交互）。
* **数据库：** PostgreSQL 16+，鼓励尝试使用 Vector Store（向量存储）。
* **安全：** Spring Security + JWT 实现无状态鉴权。
* **AI 集成：** 必须实现 **Prompt 模板管理**（不要硬编码提示词）。
    * 必须处理 **流式输出 (SSE/WebSocket)**，提升 AI 响应的用户体验。

### 2. 前端（Vue3 / React / Next.js + TypeScript）
* **工程化：** Vite + Pinia + TypeScript。
* **交互体验：** 必须包含一个 **类 ChatGPT 的对话组件**；使用 Markdown 渲染器展示 AI 返回的内容。
* **性能优化：** 实现组件的异步加载，对 AI 请求进行 Loading 状态处理。

### 3. 部署与 DevOps（加分项 🌟）
* **前端部署：** 部署至 **Vercel** 或 **Netlify**（配置自动化构建流水线）。
* **后端部署：** 部署至 **Render**、**Railway** 或 **Zeabur**，并配置环境变量管理 API Key。
* **容器化：** 提供 `docker-compose.yml` 一键运行全栈环境。
